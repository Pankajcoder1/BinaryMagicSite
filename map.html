  1. About map container
    STL container which store data in key value pair where key will be unique in nature. By default keys are sorted in ascending order. Internally it is implemented via tree and that's why it can do insertion and other operation in O(logn) time.
    Main operation on map container
    • lower_bound(k): Return an iterator of first pair whose key >= 'k' if such pair exist otherwise return key as 'size of map' with value '0'.
    • upper_bound(k): Return an iterator of immediate next pair whose key is greater than 'k' if such pair exist otherwise return key as 'size of map' with value '0'.
    • erase(): erase element from map if present otherwise do nothing. Their is two version of this operation.
      1. erase(key) -> erase key-value pair with given key.
      2. erase(iterator) -> erase key-value pair pointed by given iterator.
    • find(k): Return an iterator of a key-value pair where key = 'k' in the map if found, else return 'end' iterator.
    • size(): return size of map.
    • empty(): check if map is empty or not, return boolean value accordingly.
    Difference between map and unordered map ?
    1. map:
      • Uses a self-balancing binary search tree
      • Stores key-value pairs in a sorted order
      • Logarithmic time complexity for operations (O(log n))
      • Keys are always sorted, by default in ascending order.
    2. unordered_map:
      • Uses a hash table
      • Does not maintain any particular order
      • Constant time complexity for operations on average (O(1))
      • Faster lookup times but no specific order guaranteed
