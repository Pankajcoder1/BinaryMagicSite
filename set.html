  1. About set container
    The STL set container is a powerful data structure in C++ that stores a collection of unique elements in a sorted order. It offers insertion, deletion, and search operations with log(n) complexity and internally it is inplemented via RB tree.
    Main operation on set container
    • lower_bound(ele): Return an iterator of first element whose value is >= 'ele' if such element exist otherwise return size of container.
    • upper_bound(ele): Return an iterator of immediate next element whose value is greater than 'ele' if such element exist otherwise return size of container.
    • erase(): erase element from set if present otherwise do nothing. Their is two version of this operation.
      1. erase(ele) -> erase element with given value.
      2. erase(iterator) -> erase element pointed by given iterator.
    • find(ele): Return an iterator of a element whose value is = 'k' in the container if found, else return 'end' iterator.
    • size(): return size of set.
    • empty(): check if set is empty or not, return boolean value accordingly.
    Difference between set and unordered set ?
    1. set:
      • Uses a self-balancing binary search tree
      • Stores element in a sorted order
      • Logarithmic time complexity for operations (O(log n))
      • Elements are always sorted, by default in ascending order.
    2. unordered_set:
      • Uses a hash table
      • Does not maintain any particular order
      • Constant time complexity for operations on average (O(1))
      • Faster lookup times but no specific order guaranteed
